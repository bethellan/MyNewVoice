================================

Legacy version note files consolidated.

--- V10_CENTERED_SUBMENU_ROWS.txt ---
MyNewVoice v10 - Centered submenu rows

Change summary:
- Vertically centered the left image/icon and right phrase text within each large submenu row button.
- Preserved stacked submenu layout, locked title bar, category colours, phrase popup, recorded voice fallback, device generated speech, photo setup and backup/import features.
- Updated service-worker cache name to force refreshed CSS after GitHub upload.


--- V11_IMAGE_CROPPER.txt ---
MyNewVoice v11 - Image cropper

Changes:
- Added a crop step after choosing a local photo in Settings -> Photo / Voice Setup.
- Photos can be cropped before being saved locally on the device/browser.
- Square and rectangle crop modes are available in the crop screen.
- Phrase images default to square crops.
- Top menu and zoom images default to rectangle crops.
- Saved images are rendered with object-fit cover/contain as already appropriate, so images should not be stretched.
- Submenu phrase rows and left image slots were made moderately taller so cropped photos are easier to see.

Private photos and recorded voices remain local to the device/browser unless exported through the private backup tool.


--- V12_IOS_TTS_NO_SUBTITLES.txt ---
MyNewVoice v12 - iPhone text-to-speech fallback and popup cleanup

Changes:
- Fixed generated device speech fallback on iPhone/iPad by avoiding an async IndexedDB lookup before speechSynthesis when no local recording is known.
- If a phrase has a recorded voice, the recording still plays first.
- If no recorded voice exists, device text-to-speech starts immediately from the tap event.
- Removed subtitles UI and subtitle overlay.
- Removed the X button from the phrase popup.
- Phrase popup now closes 2 seconds after recorded or generated speech finishes.
- Existing photos, cropper, local voice recording, backup/import, menu order, and layout are retained.


--- V13_PASSWORD_COMPACT_PHOTO_VOICE_SETUP.txt ---
MyNewVoice v13 - Password-protected compact Photo / Voice Setup

Changes:
- Photo / Voice Setup is now behind the existing management password.
- Photo / Voice Setup now uses a section picker plus a collapsed item list and one selected-item control panel, rather than repeating full controls for every phrase.
- SAVE CHANGES closes the setup screen and returns to the main app.
- My People phrase images use a fixed circular crop.
- Other phrase images use a fixed square crop.
- Top menu images use a fixed square crop.
- Zoom images use a fixed rectangle crop.
- Shape options are no longer shown to the user.
- The photo picker accepts image files from the device. On iPhone/Android this allows the native photo/camera chooser where the browser supports it.
- The Main menu button inside submenus is smaller.
- Settings/theme/about icons are visible on the main screen only and hidden inside submenus.


--- V14_CARER_CONTENT_SETUP.txt ---
MyNewVoice v14 - Carer Content Setup

--- V15_SAFE_REFRESH_SETTINGS_ONLY.txt ---
MyNewVoice v15 - Safe Refresh and Settings-only Main Controls

--- V17_CLEAR_IMAGES_AUDIO.txt ---
MyNewVoice v17 - separate clear images/audio maintenance

Added password-protected Settings actions:
- Remove all images
- Remove all audio

Each action has a two-step confirmation prompt because it is destructive.
Removing images keeps recorded voices, text, menu structure, and phrase setup.
Removing audio keeps images, text, menu structure, and phrase setup; phrases fall back to device speech.
Export a complete backup before using either action if the current setup needs to be preserved.

v114 - Locked main viewport after Settings (settings-screen-refactor)

Built from: v112 emergency restore.

Root cause of screen jump / freeze bug:
- showSettingsEntryOverlay() added settings-entry-active to body, which altered CSS layout rules.
- hideSettingsOverlay() called mnvHardRestoreAfterSettingsDashboard() which forcibly cleared inline
  styles on body, html, and many app elements; this caused a reflow jump.
- mnvRestoreViewportAfterSettings() called window.scrollTo() and mutated body.style.scrollBehavior,
  triggering unnecessary iPhone viewport recalculations.
- A MutationObserver on document.body (subtree:true, attributes:true) re-ran on every DOM change
  and re-added settings-dashboard-active to body inside its callback, creating a feedback loop.
- Two delegated document.addEventListener('click') listeners fired on every click site-wide and
  called multiple setTimeout restore functions, sometimes re-triggering after the menu had
  already stabilised.
- body.modal-open { overflow-y: hidden } caused the scrollbar to appear/disappear, changing
  body clientWidth and triggering layout shift on the underlying app screen.
- 100vw sizing on .management-overlay and some settings panels included the scrollbar gutter,
  causing a 15px+ horizontal nudge on desktop and overflow on iOS.

What was removed:
- restoreMainMenuLayoutAfterModal() function and all call sites.
- mnvHardRestoreAfterSettingsDashboard() function and all call sites.
- mnvCaptureViewportBeforeSettings() function and all call sites.
- mnvRestoreViewportAfterSettings() function and all call sites.
- mnvSettingsSavedViewport variable (no longer needed).
- setupSettingsDashboardObserverV110 MutationObserver (subtree body observer).
- Two broad document.addEventListener('click') layout-restore delegated listeners.
- window.addEventListener('pageshow', restoreMainMenuLayoutAfterModal).
- window.addEventListener('orientationchange') layout restore.
- document.body.classList.add/remove('settings-entry-active', 'settings-dashboard-active').
- document.documentElement.classList.add/remove('settings-dashboard-active').
- body.style.scrollBehavior mutations.
- firstButton.focus() in showManagementPanel (could trigger iOS viewport scroll).
- CSS: body.modal-open / body.settings-open / body.dialog-open / body.crop-open overflow-y: hidden.
- CSS: body.settings-entry-active and body.settings-dashboard-active layout rules.
- CSS: body:not(.settings-dashboard-active):not(.settings-entry-active) conditional rules.
- CSS: 100vw on .management-overlay.show .management-panel replaced with 100%.
- CSS: min-width: 100vw on .management-overlay.show .management-panel replaced with min-width: 0.

What was changed:
- showSettingsEntryOverlay(): removed body class mutation and viewport capture/restore.
- hideSettingsEntryOverlay(): removed viewport restore call.
- hideSettingsOverlay(): removed both restore function calls.
- showManagementPanel(): removed setTimeout firstButton.focus() to prevent iOS scroll.
- CURRENT_APP_VERSION: v112 -> v114.
- OFFLINE_CACHE_NAME: mynewvoice-offline-v112 -> mynewvoice-offline-v114.
- sw.js CACHE_NAME: mynewvoice-v112-emergency-restore -> mynewvoice-v114-settings-screen-refactor.
- app-version.json, index.html About version text, BUILD_VERSION.txt: all updated to v114.
