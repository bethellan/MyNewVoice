MyNewVoice v111 Code Audit
==========================

Scope
-----
Reviewed the v110 package for the freezing/size-jump issue after pressing the Settings cog and after entering the password-protected Settings dashboard.

Main findings
-------------
1. Settings had accumulated several generations of emergency layout patches.
   - v108 modal layout restore code.
   - v109 Settings cog viewport restore code.
   - v110 Settings dashboard hard restore code.
   - Multiple timer-based close handlers.
   - A MutationObserver watching body/subtree class and style changes.

2. The MutationObserver and delegated click handlers were high-risk.
   They watched broad document changes and repeatedly added/removed classes or restored layout. This can cause repeated layout recalculation, screen jumps, and freezing on iPhone Safari/PWA.

3. Settings entry and Settings dashboard were still modal-style overlays.
   They attempted to protect the underlying Dad-facing screen by modifying/restoring body/html styles. That is fragile on iPhone because Safari recalculates viewport dimensions as bars, keyboard, and fixed overlays appear.

4. The internal ZIP folder name still referred to v99 while the app had advanced to v110.
   That made package inspection confusing and broke the desired naming convention.

v111 correction
---------------
1. Removed the accumulated v108-v110 JavaScript patch layer:
   - No mutation observer.
   - No modal-close re-render.
   - No timer-based layout restore.
   - No body/html style clearing loop.
   - No scroll-position repair loop.

2. Settings now opens as a contained full-screen screen:
   - Settings entry screen is fixed to the viewport.
   - Password-protected Settings dashboard is fixed to the viewport.
   - Each Settings screen scrolls internally.
   - The Dad-facing screen behind it is not resized, transformed, padded, or re-rendered.

3. CSS now defines one Settings screen contract:
   - #settingsEntryOverlay and #settingsOverlay use fixed inset: 0.
   - Width is 100%, not 100vw.
   - Height uses 100dvh.
   - Panel scrolls internally.
   - No modal-open/body width compensation.

4. Version and file naming were aligned:
   - ZIP: MyNewVoice_v111_settings_screen_refactor.zip
   - Internal folder: MyNewVoice_v111_settings_screen_refactor
   - app-version.json: v111
   - CURRENT_APP_VERSION: v111
   - Service worker cache: mynewvoice-v111-settings-screen-refactor
   - Offline cache: mynewvoice-offline-v111

Preserved
---------
- Backup/export schema.
- Phrase IDs.
- Category IDs.
- Media IDs and storage keys.
- Local image/audio behaviour.
- Speech fallback behaviour.
- Dad-facing menu content and phrase content.
- Import/export logic.

Manual QA required
------------------
1. Open main menu.
2. Tap cog.
3. Close Settings entry screen.
4. Confirm main menu dimensions are unchanged.
5. Tap cog again.
6. Choose Settings.
7. Enter password.
8. Close Settings dashboard.
9. Confirm main menu dimensions are unchanged.
10. Repeat from a submenu.
11. Confirm no freezing while scrolling menus.
12. Confirm Export Backup and Import Backup still operate.
